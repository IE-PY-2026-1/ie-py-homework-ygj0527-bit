    * **적용 내용:** 이번 파이썬 프로젝트가 주어졌을때부터 만들고 싶어졌던 **'스마트 캠퍼스 좌석 예매 시스템'**을 구상하고 설계해봄. 1차 과제여서 제어문 사용 불가라는 제약사항이 존재하지만 데이터 구조를 설계하는 중요한 단계이므로 열심히 수행하고 구성할 예정, 직접 구상한 프로그램을 거듭해서 수정해가며 창의적인 개발자로 성장하는 발판이 되길 바람.
      
 2. **내용 2 : 데이터 흐름 설계 및 변수 데이터 구조 모델링**
    * **프롬프트 요약:** "1차 과제 조건(변수5개, 자료형 3)을 나의 프로젝트에 맞게 모델링 부탁해 또한 조건문/반복문 없이 입실/퇴실 시간 차이를 계산하는 로직을 변수와 연산자 모델링해줘"
    * **적용 내용:** 변수는 최소 8개가 필요하며 (입실시간, 이용 예정 시간, 퇴실 예정 시간)은 정수 형태의 데이터를 입력 받고, (도서관 이름, 예매자 성함, 학번, 열람실 구역, 희망 좌석 번호)는 문자열 데이터를 입력 받음
    if문, 조건문이 없이 계산할 때 가장 큰 문제는 시간의 단위가 다르다는 것인데 모든 시간을 분으로 통일하고 시간 차이 계산을 통하여 잔여 시간을 계산 가능하게 함. 시간와 분을 분리하는 방법으로는 몫 연산자(//), 나머지 연산자(%)를 이용할 수 있음

## 🟩 [2차 과제: V1.0] 입출력 + 리스트 + 조건문 - 향후 작성 예정
### **✨2차 과제 내용:**
  * 본 프로젝트는 '사용자 중심의 지능형 열람실 예약 및 학습 매니지먼트 시스템' 개발을 목표로 합니다. 단순히 좌석을 선점하는 기능을 넘어, 이용자의 학습 목표(과목)를 설정하고 입실부터 퇴실까지의 전 과정을 수치화하여 관리할 수 있는 통합 솔루션을 지향합니다.
  * **데이터 및 자료형 활용** : 프로그램 초기 단계에서 시스템 환경 변수(혼잡도, 버전, 관리자 정보 등)를 문자열(str), 정수(int), 실수(float) 등 다양한 자료형으로 선언하여 시스템의 기초를 설계했습니다. 특히 학번과 같은 고유 식별자는 정수형으로, 혼잡도와 같은 유동적 지표는 실수형으로 처리하여 데이터의 정밀도를 높였습니다.
  * **알고리즘 및 가중치 연산** : 이용자의 편의성을 고려하여 '시'와 '분'을 별도로 입력받는 구조를 채택했습니다. 내부 로직에서는 이를 **분 단위 가중치 연산(Total\ Minutes = Hour \times 60 + Minute)**을 통해 단일 수치로 통합하여 시간 차 계산의 논리적 오류를 방지했습니다. 또한, 목표 학습 시간 산출 시 실수형 나눗셈을 적용하여 데이터의 연속성을 확보했습니다.
  * **다중 계층 선택 시스템** :if-elif-else를 이용한 **계층형 선택 구조(Hierarchical Selection)**를 통해 다층 구조의 도서관 공간을 효율적으로 탐색하도록 설계했습니다. 특히 and/or/not 등의 논리 연산자를 복합적으로 구성하여, 좌석 번호의 유효 범위 검증과 기존 예약 데이터와의 중복 여부를 동시에 판정하는 고도화된 예약 승인 로직을 구현했습니다.
  * **리스트 등적 관리** : append, sort, len, max 등 리스트 조작 메소드를 전방위적으로 활용하여, 실시간으로 추가되는 예약 정보를 동적으로 업데이트하고 전체 통계(총 예약자 수, 마지막 예약 번호 등)를 산출하는 기능을 포함합니다.
   
### **🤖 AI 파트너십 과정**

1. **내용 1: 도서관 공간 구조에 따른 중첩 조건문 설계 자문**
    * **프롬프트 요약:** "명지대학교 도서관의 실제 층별 구조를 반영하고 싶어. 사용자가 층수를 고르면 그 하위 열람실 목록이 나타나는 직관적인 메뉴 시스템을 조건문으로 어떻게 구현할까?"
    * **적용 내용:** AI와의 협업을 통해 **'Depth 1: 층별 클러스터링(Cluster)'**과 **'Depth 2: 구역별 상세 선택(Section)'**으로 이어지는 중첩 if 구조를 확립했습니다. 이를 통해 사용자는 자신의 학습 환경(노트북 사용 여부 등)에 맞는 최적의 공간을 시행착오 없이 선택할 수 있게 되었습니다.

2. **내용 2: 반복문과 카운트 변수를 활용한 데이터 검증 로직 최적화**
    * **프롬프트 요약:** "과제 조건인 float 형변환을 억지로 넣지 않고 기능적으로 의미 있게 쓰고 싶어. 시간 데이터를 다루면서 실수를 활용할 수 있는 계산식이 있을까?"
    * **적용 내용:** 시간(H)과 분(M)을 별도로 수집한 뒤 이를 소수점 형태의 시간 단위로 치환하는 float(H + M/60) 수식을 도입했습니다. 이는 사용자로부터 직관적인 정수 입력을 유도하면서도, 시스템 내부적으로는 정밀한 실수 데이터를 관리하게 함으로써 인터페이스 친절도와 기술적 요구사항을 동시에 충족시켰습니다.
    
### **🛠️ Troubleshooting & 기술 회고:**

1. **문제 1: 입력 데이터의 엄격한 자료형 구분 (TypeError)**
    * **원인:** input() 함수는 모든 데이터를 문자열(Unicode String)로 수신하기 때문에, 학번 합계나 이용 시간 차이 계산 시 TypeError가 발생하며 프로그램이 강제 종료되었습니다.
    * **해결:** 각 데이터의 성격에 맞춰 **명시적 형변환(Explicit Type Conversion)**을 수행했습니다. 식별자는 int(), 정밀한 수치는 float()로 래핑(Wrapping)하여 산술 연산의 안정성을 확보했으며, 이 과정에서 파이썬의 동적 타이핑 특성과 형변환의 중요성을 깊이 이해하게 되었습니다.
2. **문제 2: 논리 구조의 완결성을 위한 구문 제어 (SyntaxError)**
    * **원인:** 복잡한 조건문과 반복문이 중첩되면서 코드 블록의 시작점인 콜론(:)을 누락하거나, 파이썬 특유의 들여쓰기 규칙(Indentation Rule)이 어긋나 IndentationError 및 SyntaxError가 빈번히 발생했습니다.
    * **해결:** 코드 편집기의 린팅(Linting) 기능을 활용하여 모든 제어문 끝에 콜론 기호를 확인하고, 논리적 위계에 따른 코드 블록 정렬을 전수 점검했습니다. 특히 중첩 if문 내에서의 코드 흐름을 시각적으로 구조화하여 가독성과 실행의 완결성을 높였습니다.
