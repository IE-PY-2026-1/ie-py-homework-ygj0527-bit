## 1. 시나리오 제목

<br>


## 2. 시나리오 (5~10줄)
* (이곳에 한 학기 동안 개발할 프로그램의 전체 시나리오를 개괄적으로 요약합니다.)
* 이 프로그램은 캠퍼스 도서관 및 열람실의 '좌석 예매'을 시각화 하는 프로그램입니다.
* 우선 캠퍼스의 도서관 입구에 설치하여 사용자가 도서관을 이용하고자 할 때 도움을 줍니다. 사용자를 위해 시작 화면에 프로그램의 목적을 알리는 문구를 출력합니다. 사용자는 본인의 이름과 학번을 입력합니다. 해당 도서관의 몇 층 몇 번 좌석을 사용할 지 입력합니다. 현재 시각(입실 시간)과 퇴실 예정 시간을 입력합니다(시간/분을 구분하여 입력합니다). 입력이 완료되며 프로그램은 입력된 데이터를 바탕으로 몇 번 열람실 몇 번 좌석 예약 완료 문구와 함께 사용자가 입력한 정보들을 결과 화면에 출력합니다. 작성된 예약 확인서를 출력해 자신의 예약한 자리를 사용할 수 있게 됩니다. 도서관 사용 중 자신이 잔여 시간(n시간 n분의 형태로 확인 가능)을 확인하고 싶을 때에는 프로그램 하단에 현재 시간을 입력해 잔여 시간을 확인 할 수 있습니다.

# 🚀 [버전별 개발 일지 & AI 협업 기록]

## 🟦 [1차 과제: V1.0] 시나리오 기획
    
### **🤖 AI 파트너십 과정**
1. **내용 1 : 스마트 캠퍼스 좌석 예매 시스템 1차 구성 및 설계**
    * **프롬프트 요약:** ""나는 이번 프로젝트로 '교내 도서관 자리 예매 프로그램'을 만들고 싶은데 조건문과 반목문을 쓰지 않고 단순히 변수 선언과 입출력 함수만 사용해야 되는데 결과를 보여주는 화면을 어떻게 구성하면 좋을 지와 필요한 변수 목록과 출력 시나리오를 설계해줘.""
    * **적용 내용:** 이번 파이썬 프로젝트가 주어졌을때부터 만들고 싶어졌던 **'스마트 캠퍼스 좌석 예매 시스템'**을 구상하고 설계해봄. 1차 과제여서 제어문 사용 불가라는 제약사항이 존재하지만 데이터 구조를 설계하는 중요한 단계이므로 열심히 수행하고 구성할 예정, 직접 구상한 프로그램을 거듭해서 수정해가며 창의적인 개발자로 성장하는 발판이 되길 바람.
      
 2. **내용 2 : 데이터 흐름 설계 및 변수 데이터 구조 모델링**
    * **프롬프트 요약:** "1차 과제 조건(변수5개, 자료형 3)을 나의 프로젝트에 맞게 모델링 부탁해 또한 조건문/반복문 없이 입실/퇴실 시간 차이를 계산하는 로직을 변수와 연산자 모델링해줘"
    * **적용 내용:** 변수는 최소 8개가 필요하며 (입실시간, 이용 예정 시간, 퇴실 예정 시간)은 정수 형태의 데이터를 입력 받고, (도서관 이름, 예매자 성함, 학번, 열람실 구역, 희망 좌석 번호)는 문자열 데이터를 입력 받음
    if문, 조건문이 없이 계산할 때 가장 큰 문제는 시간의 단위가 다르다는 것인데 모든 시간을 분으로 통일하고 시간 차이 계산을 통하여 잔여 시간을 계산 가능하게 함. 시간와 분을 분리하는 방법으로는 몫 연산자(//), 나머지 연산자(%)를 이용할 수 있음

## 🟩 [2차 과제: V1.0] 입출력 + 리스트 + 조건문 - 향후 작성 예정
### **✨2차 과제 내용:**
  * 본 프로젝트는 대학 내 한정된 학습 공간인 열람실의 효율적인 배분과 이용자 편의 증진을 목적으로 합니다.
  * 단순한 정보 입력을 넘어, **층별/구역별 계층 구조**를 반영한 예약 프로세스를 구축하고, 실시간 잔여 시간 계산 기능을 통해 학습 계획 수립을 돕는 지능형 키오스크 시스템을 지향합니다.
  * 시(hour)와 분(minute) 데이터를 60을 곱하는 가중치 연산을 통해 '전체 분(Total Minutes)'으로 통합하여 계산의 정확성을 확보하고, 이를 다시 // 및 % 연산자를 사용하여 사용자 친화적인 시간 형식으로 변환 출력합니다.
  * if-elif-else 중첩 조건문을 활용하여 '층수 선택 → 세부 열람실 선택'으로 이어지는 논리적 단계를 구현함으로써 사용자 인터페이스(UI)의 직관성을 높였습니다.
  * 사용자가 입력한 퇴실 시간이 입실 시간보다 앞서는 논리적 오류를 방지하기 위해 if 연산 조건을 설정하였으며, 이미 예약된 좌석 리스트를 순회하는 for 반복문을 통해 중복 예약을 원천 차단합니다.
  * 
### **🤖 AI 파트너십 과정**

1. **내용 1: 길드 매니저 컨셉 및 로직 도출**
    * **프롬프트 요약:** "내가 길드 매니저가 되어 용사를 관리하는 프로그램을 만들 거야. 1학년 수준에서 리스트와 조건문을 활용해 등급을 매기는 재미있는 로직을 짜줘."
    * **적용 내용:** AI가 제안한 '가중치 전투력 산출' 방식과 'S등급이면서 마력이 높으면 대마법사 칭호 부여'라는 복합 조건 아이디어를 실제 코드에 반영함.

2. **내용 2: 효율적인 리스트 활용법 자문**
    * **프롬프트 요약:** "파이썬 기초 문법 단계(리스트 → 조건문 → 반복문 → 함수/딕셔너리)에 맞춰, '용사 길드 관리자' 컨셉의 프로그램이 어떻게 발전할 수 있을지 4단계 기능 확장 로드맵을 제안해줘."
    * **적용 내용:** 단순 입출력을 넘어, **'기초 정보 입력 및 리스트 저장(1차) → 전투 등급 판독기 구현(2차) → 무한 메뉴 시스템 구축(3차) → 딕셔너리를 활용한 다중 캐릭터 관리(4차)'**로 이어지는 논리적인 확장 로드맵을 확립함.
    
### **🛠️ Troubleshooting & 기술 회고:**
(예시) 
1. **문제 1: 숫자를 입력했는데 계산이 안 되는 현상 (TypeError)**
    * **원인:** `input()`으로 받은 데이터는 기본적으로 '글자'라서 숫자와 곱셈 연산이 불가능했음.
    * **해결:** `int()`와 `float()` 함수를 사용하여 입력값을 숫자로 바꾸는 '형변환' 마법을 부려 해결함.
2. **문제 2: 조건문 뒤에 자꾸 생기는 빨간 줄 (SyntaxError)**
    * **원인:** `if`와 `elif` 조건식 끝에 마법의 기호인 `:`(콜론)을 자꾸 빠뜨려서 발생함.
    * **해결:** 파이썬의 모든 조건문과 반복문 끝에는 항상 땡땡(`:`)이 들어가야 한다는 규칙을 학습하고 수정함.
